## 备份存储

备份存储后端用于存放备份过程中产生的元数据信息和备份任务日志等，对应的资源对象为 `BackupStorageLocation`

### create

[BackupStorageLocation CR](https://raw.githubusercontent.com/vmware-tanzu/velero/main/pkg/apis/velero/v1/backupstoragelocation_types.go) 

```shell
$ velero backup-location create
```

可选的 flags 包括

| 名称                   | 含义                                                         |
| ---------------------- | ------------------------------------------------------------ |
| --access-mode          | 访问权限，默认为 `ReadWrite`，可选值有 `ReadWrite` 和 `ReadOnly` |
| --backup-sync-period   | 备份同步的时间间隔，默认为 `1m`，设置为 `0s` 时表示禁用同步  |
| --bucket               | 存储后端中的 bucket 信息，最终会以目录层级的形式表示         |
| --cacert               | 连接对象存储时的 TLS 证书                                    |
| --config               | 描述存储后端的配置信息，格式为 key-value                     |
| --credential           | 存储后端认证所需要的信息，格式为 key-value，key 为 K8s secret 的名称，value 为 secret 中的 key |
| --default              | 是否为默认的存储后端                                         |
| --label-columns        | 以 `table` 格式输出时，用于自定义表格栏信息                  |
| --labels               | 设置 backup CR 对象的 label 信息                             |
| --output               | dry run，指定生成配置文件的格式，可选的有 `table`，`json` 和 `yaml` |
| --prefix               | 存储后端中的 bucket 层级下的信息，最终会以目录层级的形式表示 |
| --provider             | 存储后端 provider 信息，例如 `aws`, `gcp` 等                 |
| --show-labels          | 以 `table` 格式输出时，最后一栏展示标签信息                  |
| --validation-frequency | 存储后端同步的时间间隔，默认为 `1m`，设置为 `0s` 时表示禁用同步 |

## 即时备份

即时备份（on-demand）也就是单次的备份任务，对应的资源对象为 `Backup`。

### create

[Backup CR](https://raw.githubusercontent.com/vmware-tanzu/velero/release-1.6/pkg/apis/velero/v1/backup.go)

```shell
$ velero backup create <name>
```

可选的 flags 包括

| 名称                                        | 含义                                                         |
| ------------------------------------------- | ------------------------------------------------------------ |
| --default-volumes-to-restic                 | 默认为 `true`，即由 restic 备份所有的 Pod 卷，优先级小于 velero install 中的对应参数 |
| --include-namespaces / --exclude-namespaces | 显式包含/排除的命名空间，支持逗号分割                        |
| --include-resources / --exclude-resources   | 显式包含/排除的资源，支持逗号分割                            |
| --from-schedule                             | 基于某一个定时备份创建一次即时备份<br />指定此参数时，其他的 filter flag 均会失效，并以 schedule 的模板为准；不指定备份名称时会以 `<schedule name>-<timestamp>` 作为备份任务名称 |
| --include-cluster-resources                 | 默认为 `true`，即包含集群级别的资源<br />即使开启了此特性，如果并未备份全量 namespaces 的资源，仍然不会备份集群级别资源 |
| --label-columns                             | 以 `table` 格式输出时，用于自定义表格栏信息                  |
| --labels                                    | 设置 backup CR 对象的标签信息                                |
| --ordered-resources                         | 指定备份的顺序<br />集群级别的资源格式为 `resource name`，非集群级别的资源格式为 `namespace/resource name`，例如 pods=ns1/pod1,ns1/pod2;persistentvolumeclaims=ns1/pvc4,ns1/pvc8 |
| --output                                    | dry run，指定生成配置文件的格式，可选的有 `table`，`json` 和 `yaml` |
| --selector                                  | 默认为全量，即备份全量资源，可以通过标签选择进行筛选备份     |
| --show-labels                               | 以 `table` 格式输出时，最后一栏展示标签信息                  |
| --snapshot-volumes                          | 默认为 `true`，即备份时，默认会对 PV 资源调用对应的 provider 打快照 |
| --storage-location                          | 存储备份的后端，仅支持单个                                   |
| --ttl                                       | 过期时间，默认 `720h`                                        |
| --volume-snapshot-locations                 | 卷快照的存储后端，支持多个                                   |
| --wait                                      | 阻塞直至操作备份状态不再是 New 或者 InProgress<br />手动退出等待可以 ctrl-c，并不会影响到备份任务 |

### delete

velero backup 的删除涉及到存储在远端数据的同步删除，因此并非单纯删除 backup 对象，而是借助于 `DeleteBackupRequest` 对象。

[DeleteBackupRequest CR](https://raw.githubusercontent.com/vmware-tanzu/velero/release-1.6/pkg/apis/velero/v1/delete_backup_request.go)

```shell
$ velero backup delete
```

可选的 flags 包括

| 名称       | 含义                       |
| ---------- | -------------------------- |
| --all      | 删除所有的备份             |
| --confirm  | 确认删除交互               |
| --selector | 删除满足标签选择的所有备份 |

name，--all 和 --selector 仅能指定一个

### describe

```shell
$ velero backup describe
```

可选的 flags 包括

| 名称                       | 含义                                     |
| -------------------------- | ---------------------------------------- |
| --cacert                   | TLS 连接时的验证 bundle 的路径地址       |
| --details                  | 更详细的信息输出                         |
| --insecure-skip-tls-verify | 跳过 tls 验证                            |
| --selector                 | 可以通过标签选择器获取符合要求的备份详情 |

### download

[DownloadRequest CR](https://raw.githubusercontent.com/vmware-tanzu/velero/release-1.6/pkg/apis/velero/v1/download_request_types.go)

```shell
$ velero backup download
```

可选的 flags 包括

| 名称                       | 含义                                                        |
| -------------------------- | ----------------------------------------------------------- |
| --cacert                   | 建立 TLS 连接时所需的证书文件地址                           |
| --force                    | 下载文件存在则覆盖                                          |
| --insecure-skip-tls-verify | 跳过 TLS 验证                                               |
| --output                   | 文件保存的路径，默认为当前目录，名称为 `<name>-data.tar.gz` |
| --timeout                  | 等待下载的超时时间，默认为 1m                               |

### get

```shell
$ velero backup get
```

可选的 flags 包括

| 名称            | 含义                                                         |
| --------------- | ------------------------------------------------------------ |
| --label-columns | 在 table 样式下，自定义表格栏                                |
| --output        | 格式化输出的样式，可选的有 table，json 和 yaml，默认为 table |
| --selector      | 可以通过标签选择器展示符合要求的备份                         |
| --show-labels   | 在 table 样式下，最后一栏展示标签信息                        |

### logs

```shell
$ velero backup logs
```

可选的 flags 包括

| 名称                       | 含义                                |
| -------------------------- | ----------------------------------- |
| --cacert                   | 建立 TLS 连接时所需的证书文件地址   |
| --insecure-skip-tls-verify | 跳过 TLS 验证                       |
| --timeout                  | 等待获取 logs 的超时时间，默认为 1m |
